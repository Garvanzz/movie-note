# 添加影片弹窗快速建档设计

日期：2026-05-25

## 背景

当前“添加影片”弹窗包含两个输入项：番号和标题。这个结构会让职责变得模糊：

- 如果它只是快速建档入口，那么标题字段会让人误以为这里应该录完整资料。
- 如果它是完整录入入口，那么当前字段又远远不够，最终会和详情页职责重叠。

现有产品已经具备独立的影片详情页和刮削入口，因此新增影片的最合理定位应该是“先创建壳记录，再进入后续整理流程”。

## 目标

1. 明确“添加影片”弹窗只负责快速建档。
2. 弹窗主输入只保留番号，降低录入门槛。
3. 创建成功后让用户可以继续在详情页完善资料。
4. 避免弹窗演变成第二个详情页。
5. 与现有刮削入口、详情页编辑能力形成清晰分工。

## 非目标

1. 不在本期把“添加影片”弹窗改造成完整表单。
2. 不把标签、演员、文件、评分、备注等整理动作搬进弹窗。
3. 不改变影片详情页作为完整整理入口的定位。

## 方案选择

采用“极简快速建档”方案：

- 弹窗只保留番号输入。
- 点击添加后直接创建影片记录。
- 创建成功后关闭弹窗，并将用户带到新影片详情页继续整理。

与“继续保留标题可选项”相比，这个方案的优势是职责最清晰。与“扩展为完整录入弹窗”相比，它更符合当前项目已有的信息架构。

## 交互设计

### 弹窗内容

弹窗保留以下内容：

- 标题：添加影片
- 主字段：番号
- 操作按钮：取消、添加

移除：

- 标题（可选）输入框

### 提交流程

1. 用户输入番号。
2. 点击“添加”或回车提交。
3. 成功后：
   - 创建记录
   - 刷新影片列表缓存
   - 关闭弹窗
   - 跳转到新影片详情页
4. 失败后：
   - 保持弹窗打开
   - 显示错误提示

### 已存在影片提示

保留当前番号联想能力，用于提醒用户同番号影片可能已存在。该能力只做提示，不改变“快速建档”定位。

## 信息架构边界

新增影片后的三段职责划分如下：

- 添加影片：只负责创建最小记录
- 刮削：负责自动补充外部资料
- 详情页：负责手动整理和补全信息

这样用户路径会变成：

- 快速新增：输入番号 -> 添加 -> 进入详情页
- 自动整理：输入番号 -> 刮削导入
- 手动整理：在详情页补标题、评分、备注、标签、演员、文件

## 成功后的跳转策略

默认采用“添加后进入详情页”。

原因：

1. 新增后最常见的下一步就是继续整理。
2. 这能自然承接详情页已有编辑能力。
3. 这比留在列表页更符合“刚建档，继续完善”的操作路径。

如果后续用户需要更高频的批量快速入库，再考虑补充“添加并继续留在列表页”的第二动作，但不在本期范围内。

## 验收标准

1. 弹窗中只保留番号输入。
2. 用户可以通过按钮或回车创建影片。
3. 创建成功后自动进入新影片详情页。
4. 创建失败时保留弹窗上下文，并显示错误提示。
5. 当前刮削入口和详情页能力不受影响。

## 风险与取舍

1. 去掉标题输入后，少数想顺手补标题的用户会多一步跳转成本。
2. 但这个取舍换来了职责清晰，避免弹窗和详情页边界继续混乱。
3. 如果后续观察到“只补标题”确实是高频需求，再考虑做折叠式补充字段，而不是回到完整弹窗方案。

## 建议实施顺序

1. 修改 AddMovieDialog，只保留番号字段。
2. 提交成功后跳转到新影片详情页。
3. 保留现有联想与重复提醒能力。
4. 验证新增、失败提示、跳转流程是否正常。
